§ 0 Arquitectura General del Sistema
Stack tecnológico

🖥️ Frontend

  • React 18 + Vite
  • React Router v6
  • Leaflet (mapas)
  • Desplegado en Vercel
  • Autenticación JWT en localStorage

⚙️ Backend

  • Node.js + Express 4.18
  • Puerto 3001 (configurable)
  • Helmet + CORS + Rate limiting
  • Validación con Zod
  • Multer (archivos, máx. 50 MB)

🗄️ Base de datos

  • PostgreSQL 13+ con PostGIS
  • Supabase (cloud)
  • Schemas: raw / staging / public
  • Pool de conexiones (máx. 10)
  • ExcelJS para exportaciones

🔐 Seguridad

  • JWT Access Token (24h)
  • Refresh Token (7 días)
  • bcrypt para contraseñas
  • Rate limit: 100 req/15 min
  • Rate limit auth: 5 intentos/15 min
Flujo de gestión de datos
Cada archivo subido puede seguir el flujo legacy (inmediato) o el pipeline ETL completo (con control de calidad):
📁 Archivo
.csv/.xlsx/.geojson
→
POST /api/uploads
Registro + análisis
→
raw.upload_{id}
JSONB sin tipo
→
staging.stg_{jobId}
Tipado + georref
→
public.{tabla}
Producción + mapa
Schema raw: datos originales sin transformar  |  Schema staging: transformados, georreferenciados y validados  |  Schema public: producción, visibles en el mapa
Roles y permisos

§ 1 Autenticación /api/auth
Gestión de sesiones mediante JWT. El access token tiene vigencia de 24 horas y el refresh token de 7 días. Se recomienda almacenar el token en localStorage y enviarlo en cada petición como Authorization: Bearer <token>.
POST /api/auth/login Público Iniciar sesión y obtener tokens
Body (JSON)
CampoTipoRequeridoDescripción
username string Sí Nombre de usuario (mín. 2 caracteres)
password string Sí Contraseña (mín. 4 caracteres)
Respuesta exitosa
{ "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "refreshToken": "...",
  "user": { "id": 1, "username": "admin", "role": "admin", "secretaria": null } }
⚠️ Rate limit: máximo 5 intentos fallidos por IP cada 15 minutos.
GET /api/auth/me JWT requerido Obtener perfil del usuario activo
{ "id": 1, "username": "admin", "nombre_completo": "Administrador",
  "rol": "admin", "secretaria_id": null, "email": "..." }
POST /api/auth/refresh Público Renovar access token
Body (JSON)
CampoTipoDescripción
refreshTokenstringToken de refresco obtenido al hacer login
POST /api/auth/logout JWT requerido Cerrar sesión e invalidar tokens
{ "message": "Sesión cerrada exitosamente" }

§ 3 Pipeline ETL /api/etl
El pipeline ETL controla el procesamiento de datos en 5 etapas: Extracción (raw), Transformación (tipado de columnas), Georreferenciación (creación de geometrías PostGIS), Validación (reglas de calidad) y Promoción (publicación en producción). Todos los endpoints requieren autenticación.
Estados del pipeline: legacy → subida directa | raw → extraído | processing → en proceso | staging → en revisión | validated → validado | production → publicado | error → error
POST /api/etl/process JWT requerido Iniciar procesamiento (raw → staging)
Body (JSON)
CampoTipoReq.Descripción
upload_idintegerSíID del upload a procesar (debe estar en estado raw)
geo_modeenumNocoords | join | spatial | none (default: none)
geo_configobjectNoConfig de georreferenciación según el modo elegido
column_mappingarraySíMapeo de columnas: [{source, target, type}] (mín. 1)
validation_rulesarrayNoReglas de validación: [{field, rule, params, severity}]
Objeto column_mapping
CampoTipoDescripción
sourcestringNombre original de la columna en el archivo
targetstringNombre destino en staging (formato snake_case)
typeenumTEXT | INTEGER | NUMERIC | BIGINT | DATE | BOOLEAN
Objeto validation_rules
CampoValoresDescripción
ruleenumrequired | min | max | min_length | max_length | regex | lat_range | lon_range
severityenumwarning (default) | error
lat_range/lon_rangeColombiaLat: -4.5 a 13.5  |  Lon: -82 a -65

§ 4 GeoData — Datos Geoespaciales /api/geodata
Acceso a datos de tablas PostGIS en producción. Solo tablas registradas en geo_tablas son accesibles (whitelist con caché de 5 minutos). Retorna GeoJSON estándar. Rate limit elevado: 1000 req/15 min.
GET /api/geodata/:tableName JWT requerido Obtener datos en formato GeoJSON
Query params
ParamTipoDescripción
bboxstringBounding box: minLon,minLat,maxLon,maxLat — filtra con ST_Intersects
qstringBúsqueda de texto libre (máx. 200 chars)
searchFieldsstringColumnas para búsqueda separadas por coma
simplifynumberTolerancia de simplificación de geometría (ST_SimplifyPreserveTopology)
colsstringColumnas a retornar separadas por coma
limitintegerLímite de features (default según modo, máx. 50.000)
Respuesta — GeoJSON FeatureCollection
{ "type": "FeatureCollection",
  "features": [{
    "type": "Feature",
    "geometry": { "type": "Point", "coordinates": [-76.512, 3.014] },
    "properties": { "nombre": "Luminaria LED 001", "barrio": "Centro" } }] }
GET /api/geodata/:tableName/export JWT requerido Descargar tabla completa como Excel (.xlsx)
Genera y descarga un archivo Excel con todos los datos de la tabla (sin columna de geometría). Límite: 100.000 filas. Incluye encabezados estilizados y filas alternas. El nombre del archivo es {tableName}.xlsx.
Headers de respuesta
Content-Type: application/vnd.openxmlformats-officedocument.spreadsheetml.sheet
Content-Disposition: attachment; filename="nombre_tabla.xlsx"
X-Row-Count: 1540
§ 5 Gestión de Tablas y Schemas /api/tables
Gestión administrativa de las tablas en los tres schemas de la base de datos (raw, staging, public). Las operaciones de escritura (mover, renombrar, eliminar) requieren rol admin o editor_geo.
GET /api/tables/schemas JWT requerido Listar todas las tablas por schema
{ "schemas": {
  "raw": [{ "tableName": "upload_42", "approxRows": 1543, "columnCount": 8, "sizePretty": "240 kB",
    "upload": { "nombre_archivo": "datos.xlsx", "etl_status": "raw" } }],
  "staging": [...],
  "public": [...] } }
GET /api/tables/:schema/:table/info JWT requerido Estructura de columnas de una tabla
{ "columns": [{ "column_name": "nombre", "data_type": "character varying", "is_nullable": "YES" }],
  "rowCount": 1543 }
POST /api/tables/move Admin / editor_geo Mover tabla entre schemas
CampoTipoReq.Descripción
fromSchemastringSíSchema origen: raw | staging | public
fromTablestringSíNombre de la tabla
toSchemastringSíSchema destino
toNamestringNoNuevo nombre de la tabla (opcional)
POST /api/tables/rename Admin / editor_geo Renombrar tabla
CampoDescripción
schemaSchema donde está la tabla
oldNameNombre actual
newNameNombre nuevo
DELETE /api/tables/:schema/:table Admin / editor_geo Eliminar tabla (DROP TABLE)
⚠️ Operación destructiva. Elimina permanentemente la tabla del schema especificado.
